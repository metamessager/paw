### 核心组件

#### 1. KnotAgentAdapter
**职责**: Agent 适配和消息转换

```dart
class KnotAgentAdapter {
  // 转换为标准 Agent
  Agent toStandardAgent(KnotAgent knotAgent)
  
  // 发送消息到 Knot Agent
  Future<String> sendMessageToKnotAgent(...)
  
  // 轮询任务并转换为消息
  Future<Message?> pollTaskAndConvertToMessage(...)
}
```

**特点**:
- ✅ ID 前缀 `knot_` 避免冲突
- ✅ 智能上下文构建
- ✅ 自动错误处理

**特点**:

## 🔧 技术亮点

### 1. 适配器模式

使用适配器模式实现 Knot Agent 到标准 Agent 的无缝转换。

```dart
Agent standardAgent = adapter.toStandardAgent(knotAgent);
// Knot Agent 现在可以像普通 Agent 一样使用
```

### 2. 消息回调机制

灵活的回调系统支持多个订阅者。

```dart
bridgeService.addMessageCallback((message) {
  // 处理 Knot Agent 的响应
});
```

### 3. 自动资源管理

任务完成后自动清理映射和停止轮询。

```dart
// 任务完成后自动执行
timer.cancel();
_activePolls.remove(taskId);
_adapter.clearTaskMapping(taskId);
```

## ⚠️ 注意事项

### 1. 轮询频率

当前为每 3 秒轮询一次，大量并发任务会增加网络负载。

**优化建议**:
- Phase 3 实施 WebSocket 推送
- 动态调整轮询间隔

### 2. 任务超时

长时间运行的任务可能导致持续轮询。

**优化建议**:
- 添加任务超时机制
- 提供手动取消功能

### 3. 并发控制

同一频道多个 Agent 可能导致消息顺序问题。

**解决方案**:
- 消息带时间戳排序
- UI 按时间显示
