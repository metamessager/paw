# 🎯 AI Agent Hub - UI 集成测试指南

**测试目标**: 在 AI Agent Hub UI 中添加 4 个 Mock Agent，验证所有功能正常  
**预计时间**: 1-2 小时  
**前置条件**: 4 个 Mock Agent 已启动（端口 8081-8084）

---

## 📋 测试前准备

### 1. 确认 Mock Agent 运行状态

```bash
cd /data/workspace/clawd/ai-agent-hub

# 检查所有 Agent 是否运行
./scripts/mock_agents/check_agents.sh

# 或手动检查每个端口
curl -s http://localhost:8081/health && echo " ✅ Knot-Fast"
curl -s http://localhost:8082/health && echo " ✅ Smart-Thinker"
curl -s http://localhost:8083/health && echo " ✅ Slow-LLM"
curl -s http://localhost:8084/health && echo " ✅ Error-Test"
```

**预期输出**: 所有 Agent 返回 `{"status": "ok", ...}` ✅

---

### 2. 启动 AI Agent Hub 应用

```bash
# 确保在项目根目录
cd /data/workspace/clawd/ai-agent-hub

# 启动 Flutter 应用（开发模式）
flutter run

# 或构建并运行（生产模式）
flutter build linux
./build/linux/x64/release/bundle/ai_agent_hub
```

---

## 🚀 UI 测试步骤

### Phase 1: 添加 Mock Agent (10 分钟)

#### 测试 1.1: 添加 Knot-Fast Agent ⚡

**操作步骤**:
1. 打开 AI Agent Hub 应用
2. 导航到「Agent 管理」页面
3. 点击「添加 Agent」按钮
4. 填写以下信息：

```
Agent 名称:    Knot-Fast ⚡
Agent ID:      mock_knot_29f050f5
Endpoint:      http://localhost:8081/a2a/task
API Token:     (留空或填写任意值)
描述:          快速响应测试 Agent
```

5. 点击「保存」或「添加」

**验证点**:
- ✅ Agent 成功添加到列表
- ✅ Agent 卡片显示正确的名称和描述
- ✅ Agent 状态显示为「可用」或「在线」

**失败处理**:
- 如果提示「连接失败」，检查 Mock Agent 是否运行（`curl http://localhost:8081/health`）
- 如果提示「格式错误」，检查 Agent ID 和 Endpoint 是否正确

### Phase 2: 测试基本消息发送 (20 分钟)

#### 测试 2.1: 发送消息给 Knot-Fast ⚡

**操作步骤**:
1. 在 Agent 列表中，选择「Knot-Fast」
2. 点击「发送消息」或「开始对话」
3. 输入测试消息: `"你好，测试一下基本功能"`
4. 点击「发送」

**验证点**:
- ✅ 消息立即显示在聊天界面（用户消息）
- ✅ 在 ~0.75s 内开始收到 Agent 响应
- ✅ 响应内容逐步显示（流式响应）
- ✅ 响应完整显示以下内容：
  ```
  [Knot Agent Knot-Fast] 收到您的请求：「你好，测试一下基本功能」

  这是一个模拟的 Knot A2A 响应。我已经理解了您的问题，并准备好进行测试。

  当前时间：[时间戳]
  Agent ID: mock_knot_29f050f5

  测试成功！✅
  ```
- ✅ 响应状态显示为「完成」

**失败处理**:
- 如果超时（> 5s 无响应），检查网络连接
- 如果返回错误，查看错误信息并检查 Mock Agent 日志

#### 测试 2.4: 发送消息给 Error-Test ⚠️

2. Agent: Error-Test
3. 消息: `"测试错误处理 #1"`, `"测试错误处理 #2"`, ... (每次修改编号)

**验证点**:
- ✅ 约 30% 的请求返回错误（1-2 次失败，3-4 次成功）

**失败处理**:
- 如果所有请求都成功或都失败，这可能是随机性导致的，多测试几次
- 如果 UI 崩溃或无法处理错误，记录问题并修复错误处理逻辑

#### 测试 3.3: 验证流式响应的流畅度

**操作步骤**:
1. 依次测试所有 4 个 Agent
2. 观察响应的流式效果

**验证点**:
- ✅ Knot-Fast: 响应快速，流式效果明显但快速
- ✅ Smart-Thinker: 响应中等速度，流式效果明显
- ✅ Slow-LLM: 响应慢速，流式效果非常明显（逐字显示）
- ✅ Error-Test: 成功时响应正常，失败时立即显示错误

**流式效果评分标准**:
- ⭐⭐⭐⭐⭐: 内容逐步显示，无卡顿，用户体验流畅
- ⭐⭐⭐⭐: 内容逐步显示，偶尔卡顿
- ⭐⭐⭐: 内容分块显示，卡顿明显
- ⭐⭐: 内容一次性显示，无流式效果
- ⭐: UI 卡死或崩溃

- ✅ 重新启动 Agent 后，可以正常使用
